# Visual "Gist" Feature - Implementation Context

## Overview
"Visual Gist" allows users to capture a specific screen area (charts, diagrams, or UI elements) and send it to Gemini 2.5 Flash for a multimodal AI explanation. This file serves as the source of truth for the technical implementation.

## 1. Trigger & Shortcuts
- **Shortcut**: `Alt + Shift + G` (Suggested/Default).
  - *Note*: Chrome often requires users to manually set this in `chrome://extensions/shortcuts` if it's an existing installation.
- **Manual Trigger**: "Visual Gist" button in the extension popup.
- **Command Name**: `capture-gist` (defined in `manifest.json`).

## 2. Communication Flow
1. **Background Script** (`background/index.ts`):
   - Listens for the `capture-gist` command.
   - Ensures the content script is injected using `chrome.scripting.executeScript`.
   - Sends `GIST_CAPTURE_START` message to the active tab.
2. **Content Script** (`content/index.ts`):
   - Receives `GIST_CAPTURE_START`.
   - Calls `mountCaptureOverlay()` via the `ShadowHost`.
3. **Capture Overlay** (`content/components/CaptureOverlay.tsx`):
   - Mounts a full-screen selection UI.
   - User drags to select an area.
   - On release, it calculates coordinates and returns a `DOMRect`.
4. **Screenshot & Crop** (`content/index.ts`):
   - Requests `CAPTURE_VISIBLE_TAB` from the background script.
   - Background calls `chrome.tabs.captureVisibleTab` (returning a full-tab DataURL).
   - Content script creates an `HTMLCanvasElement`, draws the full image, and crops it to the selection rect (accounting for `window.devicePixelRatio`).
   - Converts the cropped canvas to a Base64 string.
5. **Popover & API** (`content/components/Popover.tsx`):
   - Mounts the Gist popover in `LOADING` state.
   - Shows a thumbnail of the captured image.
   - Sends the Base64 image to the FastAPI backend.

## 3. Frontend Architecture (Shadow DOM)
- **Host**: `#gist-shadow-host` is a fixed, full-screen `100vw/100vh` container with `pointer-events: none`.
- **Z-Index**: `2147483647` (highest possible).
- **Pointer Events**: 
  - Host is `none` to allow interacting with the underlying page.
  - Interactive components (`Popover`, `CaptureOverlay`) have `pointer-events: auto`.
- **CSS Isolation**: All styles are injected into the Shadow Root as a single `<style>` block containing both Popover and Overlay styles.

## 4. Backend Implementation (FastAPI)
- **Endpoint**: `POST /api/v1/simplify`
- **Request Model** (`SimplifyRequest`):
  - `selected_text`: Optional string.
  - `image_data`: Optional Base64 string (without the `data:image/png;base64,` prefix).
  - `image_mime_type`: Default `image/png`.
- **Gemini Service** (`gemini.py`):
  - Uses `google-genai` SDK.
  - Constructs a multimodal prompt using `inline_data` for the image and `text` for the prompt.
  - Model: `gemini-2.5-flash`.

## 5. Build System Context
- **Dual Config**: Vite builds the extension core (`vite.config.ts`) and the content script (`vite.content.config.ts`) separately.
- **Race Condition Fix**: `emptyOutDir` is set to `false` in the main config to prevent it from deleting the `content.js` file generated by the other build process during development (`--watch`).
- **Styles**: CSS modules are imported with the `?inline` query to be injected as strings into the Shadow DOM.

## 6. Message Types (`messages.ts`)
- `GIST_CAPTURE_START`: Trigger the overlay.
- `CAPTURE_VISIBLE_TAB`: Request a screenshot from the background.
- `GIST_REQUEST`: Send data (text/image) to the explanation engine.
